Delta eksports

Dienas delta eksports tiek nodrošināts par iepriekšējām 14 dienām. Pašlaik pieejamo delta failu sarakstu var atrast šeit: https://static.openbeautyfacts.org/data/delta/index.txt.

Katra rinda apzīmē failu, kas ir pieejams šeit: https://static.openbeautyfacts.org/data/delta/{filename}.

Faila nosaukumā ir ietverts pirmās un pēdējās JSON failā ietvertās izmaiņas UNIX laika zīmogs, lai delta failus varētu importēt (pēc izvilkšanas) ar mongoimport alfabētiskā secībā.

Lūdzu, ņemiet vērā, ka mongoexportrakstura dēļ delta faili nevar sniegt informāciju par dzēstiem produktiem. Lai no datubāzes noņemtu izdzēstos produktus, jums būs jāimportē pilns MongoDB izgāztuves fails.

JSONL datu eksports

Visa datubāze ir pieejama arī JSONL formātā (dažreiz saukts par LDJSON vai NDJSON), kur katra rinda ir JSON objekts. Tas attēlo tos pašus datus kā MongoDB eksports. Fails ir saspiests ar gzip.

Saite
https://static.openbeautyfacts.org/data/openbeautyfacts-products.jsonl.gz

Piemērots veids, kā izmantot datubāzi, ir izmantot DuckDB — procesa analīzes rīku, kas paredzēts liela datu apjoma apstrādei dažu sekunžu laikā. Jūs varat izlasīt mūsu emuāra ierakstu , kurā mēs jūs iepazīstinām ar Open Beauty Facts datubāzes izpēti un apstrādi, izmantojot DuckDB.

Parketa datu eksports uz Hugging Face

Vienkāršota JSONL izmetes versija ir pieejama arī Parquet formātā. Konvertēšanas laikā mēs filtrējām kolonnas, kas satur dublētu informāciju, tiek izmantotas iekšējai atkļūdošanai vai vienkārši nav būtiskas lietotājiem.

Parketa formāts ir izrādījies ērts:

  • Data is organized by column, rather than by row, which saves storage space and speeds up analytics queries, i.e. you can select just the columns you care about, optimizing query performances, even on entry-level computers.
  • Highly efficient data compression and decompression, making it good for storing and sharing big datasets of any kind,
  • Supports complex data types and advanced nested data structures.

The dataset is available on Hugging Face, a collaborative Machine Learning ecosystem where developers and researchers can share models and datasets.

Saite
https://huggingface.co/datasets/openfoodfacts/product-database/resolve/main/beauty.parquet?download=true

Find more information in the Wiki, including guidelines for data reuse and example queries to get started.

OpenAPI Documentation for the JSON API

Produkta datu nolasīšanai ir pieejams arī JSON API. Šis API tiek jo īpaši izmantots mobilajā lietotnē Open Beauty Facts iPhone un Android ierīcēm.

Mēs laika gaitā uzlabojam API, taču rūpīgi cenšamies saglabāt atpakaļsaderību. Lūdzu, sazinieties ar mums , ja to izmantojat, lai mēs varētu jūs informēt par iespējamām izmaiņām.

URL, lai nolasītu datus par produktu: https://world.openbeautyfacts.org/api/v2/product/[svītrkods].json

The app for Android is developed in Kotlin.

Ņemiet vērā, ka, ja izvade ir pārāk liela, varat to filtrēt — kā aprakstīts API dokumentācijā.

  • API V3 — jauna OpenAPI dokumentācija — Darbs procesā

Meklējam palīdzību OpenAPI jomā

Mēs nepacietīgi gaidām API dokumentācijas pabeigšanu, lietošanas gadījumu pievienošanu un SDK ģenerēšanas automatizāciju. Ja esat OpenAPI pazinējs, laipni aicināti palīdzēt.

Jūs esat laipni aicināti izmantot API ražošanas gadījumos, ja vien 1 API izsaukums = 1 reāla lietotāja skenēšana. Jebkurš mēģinājums nokasīt datubāzi, izmantojot API, visticamāk, tiks bloķēts, jo pilns ikdienas eksports ir pieejams šajā pašā lapā.

Tāpat HTTP galvenes nosūtīšana kopā ar API izsaukumu ir labs veids, kā mēs varam sazināties ar jums, ja rodas kādas problēmas.

Ja kaut kas nav skaidrs vai ir kādi jautājumi (nav muļķīgu jautājumu), droši uzdodiet tos #api kanālā pakalpojumā Slack :-)

Ietvari un SDK

Mums droši vien ir apvalks jūsu iecienītākajai programmēšanas valodai. Ja mēs to darīsim, jūs varēsiet to izmantot un uzlabot. Ja mēs to nedarīsim, jūs varat palīdzēt to radīt.

Tie ļaus jums patērēt datus, kā arī ļaus jūsu lietotājiem sniegt jaunus datus.

Vispārīgi principi, lai jūsu lietotāji būtu apmierināti

  • You can search for information about products, including many useful computed values.
  • DART: GitHub (Open Food Facts only at the moment)
  • You can also implement the complete flow so that they get immediately the result with some effort on their side.

Ja lietotāji negaida tūlītēju rezultātu (piemēram, inventarizācijas lietotnes)

  • Submit photos (front/nutrition/ingredients): most painless thing for your users
  • Atvērto skaistumkopšanas faktu mākslīgais intelekts — Robotoff — no fotoattēliem ģenerēs atvasinātus datus.
  • Laika gaitā citas lietotnes un Open Beauty Facts kopiena aizpildīs datu nepilnības.

Ja lietotāji sagaida tūlītēju rezultātu (piemēram, alerģijas lietotnes)

  • Iesniegt sastāvdaļas + kategoriju > saņemt sastāvdaļu analīzi, piedevas, alergēnus, normalizētas sastāvdaļas, vegāns, veģetārietis…
  • Iesniedziet etiķetes, lai iegūtu pievienoto vērtību
