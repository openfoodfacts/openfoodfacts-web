Експорт дельти

Щоденний експорт дельти представлені на попередні 14 днів. Список доступних наразі дельта-файлів можна знайти за адресою https://static.openbeautyfacts.org/data/delta/index.txt. Кожен рядок представляє файл, який є доступним на https://static.openproductsfacts.org/data/delta/{filename}. Назва файлу містить часову позначку UNIX на першій та останній зміні, що міститься у файлі JSON, таким чином дельта файли можуть бути імпортовані (після витягнення) з mongoimport у алфебетному порядку.

Будь ласка зауважте, що через природу mongoexport, дельти файлів не можуть розповісти вам про видалені продукти. Будь ласка зауважте, що через природу mongoexport, дельти файлів не можуть розповісти вам про видалені продукти.

Діючий API JSON

API працює на 99% як Open Food Facts, тому багато SDK, коди для взаємодії з Open Food Facts API повинні працювати, просто змінюючи кінцеву точку з openfoodfacts.org на openproductsfacts.org. Код не оновлювався понад рік, тому багато новіших API, як-от панелі знань, ще недоступні.

API JSON також є наявним для читання даних продукту. Цей API зокрема використовується у мобільних додатках Open Products Facts для iPhone та Android.

Ми з часом змінюємо API, проте ми ставимося з обережністю до підтримання зворотньої сумісності. Будь ласка розкажіть нам чи ви використовуєте це, таким чином ми зможемо розповісти вам про наші потенційні зміни.

URL для зчитування даних про товар: https://world.openfoodfacts.org/api/v2/product/[barcode].json

Наприклад: https://world.openfoodfacts.org/api/v3/product/737628064502.json

  • API V1 - Стара документація доступна на Вікі (Важливе зауваження: сторінка розділена на підсторінки для ЧИТАННЯ і ЗАПИСУ).
  • API V2 - повний, але не сумісний з OpenAPI - містить варіанти використання для відображення продукту і дозволяє вашим користувачам додавати дані, необхідні для отримання Nutri-Score, класифікації NOVA, добавок та Eco-Score.
  • API V2 - нова документація OpenAPI - Work in progress
  • API V3 - нова документація по OpenAPI - Робота в процесі

Ми дуже охочі до використання API для виробничих кейсів, доки 1 API = 1 справжньому скануванню користувача. Будь-яка спроба здерти базу даних використовуючи API буде ймовірно заблокованою, так як повний щоденний експорт саме на цій сторінці.

Також, надсилання HTTP заголовку з вашим викликом API є хорошим способом для нашого зв'язку, якщо виникне будь-яке питання.

Якщо ви непевні щодо чогось, або маєте якісь запитання (немає дурних запитань), вільно почувайтеся спитати їх у Slack :-)
