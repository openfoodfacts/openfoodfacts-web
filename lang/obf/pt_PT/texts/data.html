Condições de reutilização

A base de dados do Open Beauty Facts está disponível sob a Licença Open Database.
Os conteúdos individuais da base de dados estão disponíveis sob a Licença Database Contents.
As imagens dos produtos estão disponíveis sob a Licença Creative Commons Atribuição-CompartilhaIgual. Estas podem conter elementos gráficos sujeitos a direitos de autor ou outros direitos, que em alguns casos podem ser reproduzidos (direitos de citação ou uso razoável/fair use).

Por favor, leia os Termos e condições de uso e reutilização antes de reutilizar os dados.

Fale-nos da sua reutilização

Estamos muito interessados em saber em que são usados os dados do Open Beauty Facts. Não é obrigatório, mas gostaríamos muito que nos dissesse como reutiliza os nossos dados para que possamos partilhá-los com a comunidade do Open Beauty Facts. Também pode preencher este formulário para ter a oportunidade de ter a sua aplicação em destaque.

Exportação de dados abertos do Open Beauty Facts

São geradas todas as noites dumps e exportações da base de dados. A base de dados é muito grande, mas pode encontrar várias formas diferentes de usar os dados, dependendo do tipo de exportação.

As informações sobre os diferentes campos para o dump do MongoDB e as exportações de CSV estão disponíveis em https://world-obf.openbeautyfacts.org/data/data-fields.txt

Dump MongoDB

Os dados de todos os produtos estão disponíveis num dump da base de dados MongoDB.

Hiperligação
https://static.openbeautyfacts.org/data/openbeautyfacts-mongodbdump.gz
sha256sum
https://static.openbeautyfacts.org/data/gz-sha256sum
md5sum
https://static.openbeautyfacts.org/data/gz-md5sum

Exportação delta

Estão disponíveis as exportações diárias de delta dos últimos 14 dias. A lista de ficheiros delta atualmente disponíveis pode ser encontrada em https://static.openbeautyfacts.org/data/delta/index.txt.

Cada linha representa um ficheiro que está disponível em https://static.openbeautyfacts.org/data/delta/{filename}.

O nome do ficheiro contém o carimbo de data/hora UNIX da primeira e da última alteração contida no ficheiro JSON, para que os ficheiros delta possam ser importados (após a extração) com mongoimport em ordem alfabética.

Por favor, note que, devido à natureza do mongoexport, os ficheiros delta não podem informá-lo sobre os produtos eliminados. Para remover produtos eliminados da sua base de dados, terá de importar o dump completo do MongoDB.

Exportação de dados JSONL

Toda a base de dados está também disponível no formato JSONL (por vezes chamado de LDJSON ou NDJSON), onde cada linha é um objeto JSON. Representa os mesmos dados que a exportação MongoDB. O ficheiro é comprimido com gzip.

Hiperligação
https://static.openbeautyfacts.org/data/openbeautyfacts-products.jsonl.gz

Uma forma adequada de explorar a base de dados é usar o DuckDB, uma ferramenta analítica em processo projetada para processar grandes quantidades de dados em uma fração de segundos. Pode ler a nossa publicação no blogueonde o orientamos na exploração e processamento da base de dados Open Beauty Facts com DuckDB

Exportação de dados do Parquet no Hugging Face

Também está disponível uma versão simplificada do dump JSONL no formato Parquet. Durante a conversão, filtramos colunas que contêm informações duplicadas, são usadas para depuração interna ou são simplesmente irrelevantes para os utilizadores.

O formato Parquet revelou-se útil:

  • Os dados são organizados por coluna, em vez de linha, o que poupa espaço de armazenamento e acelera as consultas analíticas, ou seja, pode selecionar apenas as colunas que lhe interessam, optimizando o desempenho das consultas, mesmo em computadores de nível básico.
  • Compressão e descompressão de dados altamente eficientes, o que o torna ideal para armazenar e partilhar grandes conjuntos de dados de qualquer tipo,
  • Suporta tipos de dados complexos e estruturas de dados agrupadas avançadas.

O conjunto de dados está disponível em Hugging Face, um ecossistema colaborativo de aprendizagem automática onde os programadores e investigadores podem partilhar modelos e conjuntos de dados.

Hiperligação
https://huggingface.co/datasets/openfoodfacts/product-database/resolve/main/beauty.parquet?download=true

Encontre mais informações na Wiki, incluindo diretrizes para reutilização de dados e consultas de exemplo para começar.

Exportação de dados CSV

Os dados de todos os produtos, ou alguns dos produtos, podem ser descarregados no formato CSV (legível com o OpenOffice, Excel e muitos outros programas) através do formulário de pesquisa avançada.

Hiperligações
Alimentos: https://static.openfoodfacts.org/data/en.openfoodfacts.org.products.csv.gz (CSV comprimido no formato GZIP: ~ 0,9 Gb, descomprimido: ~ 9 Gb)
Cosméticos: https://static.openbeautyfacts.org/data/en.openbeautyfacts.org.products.csv.gz
Comida para animais: https://static.openpetfoodfacts.org/data/en.openpetfoodfacts.org.products.csv.gz
Todos os outros produtos: https://static.openproductsfacts.org/data/en.openproductsfacts.org.products.csv.gz

A codificação do ficheiro é Unicode UTF-8. O caractere que separa os campos é <tab> (tabulação).

Exportação de dados RDF

A base de dados também está disponível no formato RDF (trata-se de uma experiência, que já não é mantida ativamente). Pode ler o anúncio em francês.

Hiperligação
https://world-obf.openbeautyfacts.org/data/en.openbeautyfacts.org.products.rdf.gz

Exportação de imagens

Todas as imagens e resultados de OCR podem ser descarregados do nosso servidor ou do AWS (através do Programa de Dados Abertos do AWS). Consulte a nossa documentação para saber mais sobre como descarregar imagens

Documentação OpenAPI para a API JSON

Também está disponível uma API JSON para ler os dados de um produto. Essa API é usada em particular na aplicação móvel Open Beauty Facts para iPhone e Android.

Atualizamos a API periodicamente, mas garantimos a compatibilidade com versões anteriores. Por favor, diga-nos se a utilizar, para que possamos informá-lo sobre possíveis alterações.

URL para ler os dados de um produto: https://world-obf.openbeautyfacts.org/api/v2/product/[barcode].json

Exemplo: https://world-obf.openbeautyfacts.org/api/v2/product/3600550578041.json

Note-se que se a saída for demasiado grande, pode filtrá-la - tal como descrito na documentação da API.

  • API V3 - nova documentação OpenAPI - Trabalho a decorrer

Procura-se ajuda OpenAPI

Estamos ansiosos para concluir a documentação da API, adicionar casos de uso e automatizar a geração do SDK. Se é um conhecedor da OpenAPI, será muito bem-vindo para ajudar.

Você é muito bem-vindo a usar a API para casos de produção, desde que 1 chamada à API = 1 verificação real por um utilizador. Qualquer tentativa de extrair dados da base de dados utilizando a API será muito provavelmente bloqueada, uma vez que as exportações diárias completas estão disponíveis nesta mesma página.

Além disso, enviar um cabeçalho HTTP com a sua chamada de API é uma boa forma de entrarmos em contacto, caso surja algum problema.

Se tiver dúvidas sobre qualquer coisa, ou se tiver alguma pergunta (não há perguntas disparatadas), sinta-se à vontade para a fazer no canal #api no Slack :-)

Wrappers e SDKs

Provavelmente temos um wrapper para sua linguagem de programação favorita. Se o fizermos, pode usá-lo e melhorá-lo. Se não o fizermos, pode ajudar a criá-lo.

Permitem que consuma dados e que os utilizadores contribuam com novos dados

Princípios gerais para deixar seus utilizadores satisfeitos

  • Pode pesquisar informações sobre produtos, incluindo muitos valores calculados úteis.
  • Se não conseguir obter informações sobre um produto específico, pode pedir ao seu utilizador que envie fotografias e dados, que serão depois processados pela IA e pelos colaboradores do Open Beauty Facts para obter o resultado calculado que pretende mostrar-lhes.
  • Pode também implementar o fluxo completo para que recebam imediatamente o resultado com algum esforço do seu lado.

Se os seus utilizadores esperam imediatamente um resultado (por exemplo, aplicações de nutrição)

  • Envie fotografias (frente/nutrição/ingredientes): o mais fácil para os seus utilizadores
  • O Open Beauty Facts AI - Robotoff - irá gerar alguns dados derivados das fotografias
  • Ao longo do tempo as outras aplicações e a comunidade do Open Beauty Facts preencherão as lacunas de dados

Se os seus utilizadores esperam imediatamente um resultado (por exemplo, aplicações de alergias)

  • Enviar ingredientes + categoria > obter a análise de ingredientes, aditivos, alergénios, ingredientes normalizados, vegan, vegetariano…
  • Enviar etiquetas para obter valor acrescentado
