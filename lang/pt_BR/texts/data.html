Exportação de dados abertos do Open Food Facts

São geradas todas as noites dumps e exportações da base de dados. A base de dados é muito grande, portanto poderá encontrar diferentes formas de usar os dados, dependendo do tipo de exportação.

As informações sobre os diferentes campos para o dump do MongoDB e as exportações de CSV estão disponíveis em https://world.openfoodfacts.org/data/data-fields.txt

Dump MongoDB

Os dados de todos os produtos estão disponíveis num dump da base de dados MongoDB.

Hiperligação
https://static.openfoodfacts.org/data/openfoodfacts-mongodbdump.tar.gz

md5sum

Exportação delta

Estão disponíveis as exportações diárias de delta dos últimos 14 dias. A lista de ficheiros delta atualmente disponíveis pode ser encontrada em https://static.openfoodfacts.org/data/delta/index.txt. Cada linha representa um ficheiro que está disponível em https://static.openfoodfacts.org/data/delta/{filename}. O nome do ficheiro contém o timestamp do UNIX da primeira e última alteração contida no ficheiro JSON, para que os ficheiros delta possam ser importados (após a extração) com mongoimport por ordem alfabética.

Por favor, note que, devido à natureza do mongoexport, os ficheiros delta não podem informá-lo sobre produtos eliminados. Para remover produtos eliminados doa sua base de dados, terá de importar o dump completo do MongoDB.

Exportação de dados JSONL

Toda a base de dados também está disponível no formato JSONL (por vezes chamado de LDJSON ou NDJSON), onde cada linha é um objeto JSON. Representa os mesmos dados que a exportação MongoDB. O ficheiro é comprimido com gzip.

Hiperligação
https://static.openfoodfacts.org/data/openfoodfacts-products.jsonl.gz

Exportação de dados RDF

Os dados de todos os produtos, ou alguns dos produtos, podem ser descarregados no formato CSV (legível com o OpenOffice, Excel e muitos outros programas) através do formulário de pesquisa avançada.

Hiperligação
https://static.openfoodfacts.org/data/en.openfoodfacts.org.products.csv

A codificação do ficheiro é Unicode UTF-8. O caractere que separa os campos é <tab> (tabulação).

API JSON ao vivo

Também está disponível uma API JSON para ler os dados de um produto. Essa API é usada em particular na aplicação móvel Open Food Facts para iPhone e Android.

Alteramos a API ao longo do tempo, mas temos o cuidado de manter a compatibilidade com versões anteriores. Por favor, diga-nos se o utilizar, para que possamos informá-lo sobre possíveis alterações.

URL para ler os dados de um produto: https://world.openfoodfacts.org/api/v0/product/[barcode].json

Exemplo: https://world.openfoodfacts.org/api/v0/product/737628064502.json

Documentação adicional está disponível na Wiki. (Nota importante: a página está dividida em sub-páginas para READ e WRITE). Está a ser desenvolvida uma melhor documentação, com casos de utilização para a visualização de um produto e permitindo que os seus utilizadores adicionem os dados necessários para obter os grupos Nutri-Score, NOVA e aditivos. Pode aceder aqui.

Pode usar a API para casos de produção, desde que 1 chamada de API = 1 varredura real de um utilizador. Qualquer tentativa de sobrecarregar a base de dados pela API provavelmente será bloqueada, pois as exportações completas diárias estão disponíveis nesta página.

Além disso, enviar um cabeçalho HTTP com a sua chamada de API é uma boa maneira de entrarmos em contacto, caso haja qualquer problema.

Se não tiver certeza de nada ou tiver alguma dúvida (não existem perguntas absurdas), fique à vontade para perguntar no Slack :-)

Aplicação para aparelhos móveis Android e iPhone

O código para a aplicação móvel Open Food Facts está disponível no GitHub (Android e iOS). A aplicação permite que os utilizadores digitalizem o código de barras dos produtos, visualizem as informações do produto, tirem fotos e enviem os dados de produtos que faltam.

A aplicação para iOS é desenvolvida no Swift.

A aplicação para Android é desenvolvida em Java.

Está convidado a reutilizar o código (é open source) e nos ajudar a melhorá-lo, para todos, no planeta inteiro.

Wrappers e SDKs

Provavelmente temos um wrapper para sua linguagem de programação favorita. Se o fizermos, pode usá-lo e melhorá-lo. Se não o fizermos, pode ajudar a criá-lo.

Permitiram que consuma dados, bem como que seus utilizadores contribuam com novos dados

Princípios gerais

  • Pode procurar informações sobre produtos, incluindo muitos valores computados úteis.
  • Se não puder obter informações sobre um produto específico, pode fazer com que seu utilizador envie fotos e dados, que então serão processados pelo Open Food Facts AI e colaboradores para obter o resultado calculado que deseja mostrar.
  • Pode também implementar o fluxo completo para que recebam imediatamente o resultado com algum esforço do seu lado.

Se os seus utilizadores não esperam resultados imediatamente (por ex. aplicações de inventário)

  • Enviar fotos (frente / nutrição / ingredientes): a coisa mais indolor para os seus utilizadores
  • O Open Food Facts AI Robotoff irá gerar alguns dados derivados das fotos
  • Ao longo do tempo, outras aplicações, e a comunidade do Open Food Facts preencherão as lacunas de dados

Se os utilizadores esperam imediatamente um resultado (por exemplo, aplicações de nutrição)

  • Envie as informações nutricionais + categoria > obtenha Nutri-Score
  • Enviar ingredientes > obter o grupo NOVA (sobre ultra-processamento de alimentos), aditivos, alergénos, ingredientes normalizados, vegano, vegetariano…
  • Envie etiquetas + categorias > e obtenha o Eco-Score (sobre o impacto ambiental)
