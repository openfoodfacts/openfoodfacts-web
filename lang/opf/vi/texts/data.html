Xuất dữ liệu kiểu Delta

Xuất dữ liệu delta hàng ngày được cung cấp cho dữ liễu trong vòng 14 ngày trước đó. The list of currently available delta files can be found at https://static.openproductsfacts.org/data/delta/index.txt. Each line represents a file that is available at https://static.openproductsfacts.org/data/delta/{filename}. Tên tệp chứa kí dấu thời gian UNIX của lần thay đổi đầu tiên và cuối cùng có trong tệp JSON, để các tệp delta có thể được nhập (sau khi trích xuất) với mongoimport theo thứ tự bảng chữ cái.

Xin lưu ý rằng do bản chất của mongoexport, các tệp delta không thể lưu trữ thông tin về các sản phẩm đã bị xóa. Để xóa sản phẩm đã xóa khỏi cơ sở dữ liệu của bạn, bạn sẽ cần nhập liệu toàn bộ tệp MongoDB.

API JSON trực tiếp

The API behaves 99% like Open Food Facts, so many of the SDKs, codes to interact with the Open Food Facts API should work by merely changing the endpoint from: openfoodfacts.org to openproductsfacts.org. The code hasn't been updated in over a year, so many of the newer APIs like Knowledge Panels are not available yet.

API JSON cũng có sẵn để đọc dữ liệu cho một sản phẩm. This API is in particular used in the Open Products Facts mobile app for iPhone and Android.

Chúng tôi thay đổi API theo thời gian nhưng chúng tôi luôn cân nhắc cẩn thận để giữ khả năng tương thích. Vui lòng cho chúng tôi biết nếu bạn sử dụng nó, để chúng tôi có thể cho bạn biết về những thay đổi có thể xảy ra.

URL để đọc dữ liệu sản phẩm: https://world.openproductsfacts.org/api/v2/product/[barcode].json

Ví dụ: https://world.openproductsfacts.org/api/v2/product/8710847992063.json

  • API V1- Old documentation available on the Wiki (Important note: the page is divided into subpages for READ and WRITE).
  • API V2 - Comprehensive but not OpenAPI compliant - It contains uses cases for displaying a product, and letting your users add the data required to get the Nutri-Score, the NOVA classification, additives, and the Eco-Score.
  • API V2 - new OpenAPI documentation - Work in Progress
  • API V3 - new OpenAPI documentation - Work in Progress

Chúng tôi rất hoan nghênh bạn sử dụng API cho các trường hợp sản xuất, miễn là 1 lần gọi API = 1 lần quét thực bởi người dùng. Bất kỳ nỗ lực nào để loại bỏ cơ sở dữ liệu bằng cách sử dụng API sẽ rất có thể bị chặn, vì các bản xuất hàng ngày đầy đủ luôn có sẵn trên chính trang này.

Ngoài ra, gửi tiêu đề HTTP với lệnh gọi API của bạn là một cách tốt để chúng tôi có thể liên hệ, nếu có bất kỳ vấn đề nào phát sinh.

Nếu bạn không chắc chắn về bất cứ điều gì hoặc có bất kỳ câu hỏi nào (không có câu hỏi ngoài lề), hãy hỏi nó trên Slack :-)
