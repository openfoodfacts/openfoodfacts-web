Делта Износ

Предоставят се ежедневни делта износи за предходните 14 дни. Списъкът с наличните в момента делта файлове може да бъде намерен на https://static.openproductsfacts.org/data/delta/index.txt. Всеки ред представлява файл, който е достъпен на https://static.openproductsfacts.org/data/delta/{filename}. Името на файла съдържа UNIX времева маркировка на първата и последната промяна, съдържащи се в JSON файла, така че делта файловете могат да бъдат внасяни (след извличане) с mongoimport по азбучен ред.

JSON API на живо

API се държи 99% като Open Food Facts, така че много от SDK, кодовете за взаимодействие с Open Food Facts API трябва да работят, като просто променят крайната точка от: openfoodfacts.org на openproductsfacts.org. Кодът не е актуализиран от повече от година, така че много от по-новите API като Knowledge Panels все още не са налични.

Наличен е и JSON API за четене на данните за продукт. Този API се използва по-специално в мобилното приложение Open Products Facts за iPhone и Android.

Променяме API с течение на времето, но внимаваме да поддържаме обратна съвместимост. Моля, кажи ни дали го използваш, за да можем да ти кажем за потенциални промени.

URL за четене на данни за продукт: https://world.openproductsfacts.org/api/v2/product/[barcode].json

Пример: https://world.openproductsfacts.org/api/v2/product/8710847992063.json

  • API V1- Стара документация, налична в Wiki (Важна забележка: страницата е разделена на подстраници за ЧЕТЕНЕ и ПИСАНЕ).
  • API V2 - Изчерпателен, но не съвместим с OpenAPI - Съдържа случаи на използване за показване на продукт и позволява на вашите потребители да добавят данните, необходими за получаване на Nutri-Score, NOVA класификация, добавки и Eco-Score.
  • API V2 - нова документация за OpenAPI - Работа в процес
  • API V3 - нова документация на OpenAPI - Работа в процес

Добре дошли сте да използвате API за производствени случаи, стига 1 API извикване да е = на 1 реално сканиране от потребител. Всеки опит за сканиране на базата данни с помощта на API много вероятно ще бъде блокиран, тъй като пълните ежедневни износи са достъпни точно на тази страница.

Освен това изпращането на HTTP хедър с вашето API извикване е добър начин да се свържем, ако възникне някакъв проблем.

Ако не сте сигурни в нещо или имате някакъв въпрос (няма глупав въпрос), не се колебайте да го зададете на Slack :-)

Мобилно приложение за Android и iPhone (необходима е помощ!)

Кодът за мобилното приложение Open Products Facts, което в момента е внедрено, е наличен в GitHub (Android. Нуждаем се от помощ, за да адаптираме новия код на Flutter за козметика Приложението позволява на потребителите да сканират баркода на продуктите, да преглеждат информацията за продукта и да правят и изпращат снимки и данни за липсващи продукти.

Приложението за Android е разработено в Kotlin.

Новото междуплатформено приложение е разработено във Flutter/Dart.

Можеш да използваш повторно кода (той е с отворен код) и да ни помогнеш да го подобрим за всички, на цялата планета.
