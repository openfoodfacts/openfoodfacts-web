Open Products Facts data exports

数据库转储和导出每晚生成。 数据库非常大，因此您可以找到 多种不同的方式来使用数据，具体取决于导出风格。

Information on the different fields for the MongoDB dump and CSV exports is available at https://world.openproductsfacts.org/data/data-fields.txt

MongoDB 转储

所有产品的数据都可以在 MongoDB 数据库转储中找到。

链接
https://static.openproductsfacts.org/data/openproductsfacts-mongodbdump.gz
SHA256 校验和
https://static.openproductsfacts.org/data/gz-sha256sum
MD5 校验和
https://static.openproductsfacts.org/data/gz-md5sum

Delta Export

提供前 14 天的每日增量导出数据。 The list of currently available delta files can be found at https://static.openproductsfacts.org/data/delta/index.txt.

Each line represents a file that is available at https://static.openproductsfacts.org/data/delta/{filename}.

文件名包含 JSON 文件中第一个和最后一个更改的 UNIX 时间戳，以便可以按字母顺序使用 mongoimport 导入增量文件（提取后）。

请注意，由于 mongoexport的性质，增量文件无法告诉您有关已删除产品的信息。 要从数据库中删除已删除的产品，您需要导入完整的 MongoDB 转储。

JSONL 数据导出

整个数据库也可以采用 JSONL 格式 （有时称为 LDJSON 或 NDJSON），其中每一行都是一个 JSON 对象。 它代表与 MongoDB 导出相同的数据。 该文件使用 gzip 压缩。

链接
https://static.openproductsfacts.org/data/openproductsfacts-products.jsonl.gz

利用数据库的一个合适方法是使用 DuckDB，这是一种旨在在几分之一秒内处理大量数据的进程内分析工具。 A suitable way to exploit the database is to use DuckDB, an in-process analytical tool designed to process large amount of data in a fraction of seconds. You can read our blog post where we walk you through exploring and processing the Open Products Facts database with DuckDB

CSV 数据导出

Data for all products, or some of the products, can be downloaded in CSV format (readable with OpenOffice, Excel and many other software) through the advanced search form.

链接
https://static.openproductsfacts.org/data/en.openproductsfacts.org.products.csv.gz (compressed CSV in GZIP format: ~ 0.9 Gb, uncompressed: ~ 9 Gb)

文件编码为Unicode UTF-8。 分隔字段的字符是 <tab> （制表符）。

实时 JSON 和 XML API

JSON API 也可用于读取产品数据。 This API is in particular used in the Open Products Facts mobile app for iPhone and Android.

我们会不断改进 API，但我们会小心保持旧版本兼容性。 Please get in touch if you use it, so that we can tell you about potential changes.

URL to read data for a product: https://world.openproductsfacts.org/api/v2/product/[barcode].json

Example: https://world.openproductsfacts.org/api/v2/product/8710847992063.json

请注意，如果输出太多，您可以对其进行过滤 - 按照 API 文档中所述。

  • API V3 - new OpenAPI documentation - Work in Progress

寻求有关 OpenAPI 的帮助

我们渴望完成 API 的文档、添加用例并自动化 SDK 生成。 如果您对 OpenAPI 十分了解，我们需要您的协助。

非常欢迎您将 API 用于生产案例，只要 1 次 API 调用 = 用户进行 1 次真实数据请求。 任何使用 API 抓取数据库的尝试都很可能被阻止，因为完整的每日导出数据就在这个页面上提供。

此外，如果出现任何问题，在 API 调用中发送您的 HTTP 标头是与我们联系的好方法。

包装器和 SDK

我们可能有您最喜欢的编程语言的包装器。 请尽量使用并改进它。 如果你没有找到，你也可以帮助创建它。

它们不仅能让你消费数据，还能让你的用户贡献新数据

让用户满意的一般原则

  • 您可以搜索有关产品的信息，包括许多有用的计算值。
  • If you can't get the information on a specific product, you can get your user to send photos and data, that will then be processed by Open Products Facts AI and contributors to get the computed result you want to show them.
  • 您还可以实施完整的流程，以便他们只需付出一些努力就能立即获得结果。

如果您的用户不期望立即获得结果（例如库存应用程序）

  • 提交照片（正面/营养/成分）：对用户来说最轻松的事情
  • The Open Products Facts AI - Robotoff - will generate some derived data from the photos
  • Over time, other apps, and the Open Products Facts community will fill the data gaps

If your users expect a result immediately (e.g. Nutrition or Allergy apps)

  • 提交成分 + 类别，然后获取成分分析、添加剂、过敏原、标准化成分、纯素食、素食等信息
  • 提交标签和它对应的内容
  • Submit nutrition
