Los volcados de la base de datos y las exportaciones se generan todas las noches.

La información sobre los diferentes campos para el volcado de MongoDB y las exportaciones de CSV está disponible en https://world.openpetfoodfacts.org/data/data-fields.txt.

Exportar datos en formato CSV

Los datos de todos los productos, o algunos de los productos, se pueden descargar en formato CSV (legible con OpenOffice, Excel y muchos otros programas de hojas de cálculo) a través del formulario de búsqueda avanzada.

Enlace
https://static.openpetfoodfacts.org/data/en.openpetfoodfacts.org.products.csv

La codificación del archivo es Unicode UTF-8. El carácter que separa los campos es <tab> (tabulación).

API JSON

La API se comporta en un 99% como la de Open Food Facts, por lo que muchos de los SDK, códigos para interactuar con la API de Open Food Facts deberían funcionar simplemente cambiando la última parte de «openfoodfacts.org» a «openpetfoodfacts.org». El código no se ha actualizado en más de un año, por lo que muchas de las API más nuevas, como los Paneles de Conocimiento, aún no están disponibles.

También está disponible una API JSON para leer los datos de un producto. Esta API se utiliza particularmente en la aplicación de Open Pet Food Facts para iPhone y Android.

Poco a poco vamos cambiando la API con el tiempo, pero procuramos mantener la compatibilidad con versiones anteriores. Si la vas a utilizar, dínoslo para que te informemos sobre posibles cambios.

URL para leer los datos de un producto: https://world.openpetfoodfacts.org/api/v0/product/[barcode].json

Ejemplo: https://world.openpetfoodfacts.org/api/v0/product/3600550578041.json

  • API V1: documentación antigua disponible en la Wiki. Nota importante: esta página está dividida en subpáginas para LEER y ESCRIBIR.
  • API V2: completa, pero no compatible con OpenAPI. Contiene casos de uso para mostrar un producto y permitir que los usuarios añadan la información necesaria para obtener el Nutri-Score, la clasificación NOVA, aditivos y el Eco-Score.
  • API V2: nueva documentación de OpenAPI. Trabajo en curso.
  • API V3:nueva documentación OpenAPI. Trabajo en curso.

Te invitamos a que utilices la API para casos de producción, siempre que 1 llamada de API = 1 escaneo real por parte de un usuario. Es muy probable que se bloquee cualquier intento scrapear la base de datos utilizando la API, ya que las exportaciones diarias completas están disponibles en esta misma página.

Además, en caso de que surja algún problema, enviar un encabezado HTTP con tu llamada API es una buena manera de ponernos en contacto.

Si tienes dudas sobre algo, o si tienes alguna pregunta (no existen las preguntas tontas), no dudes en consultarnos a través de Slack. :-)

Aplicación móvil para Android y iPhone (¡Se necesita ayuda!)

El código de la aplicación móvil de Open Pet Food Facts actualmente implementada está disponible en GitHub (Android). Necesitamos ayuda para adaptar el nuevo código Flutter a los productos alimenticios de mascotas. La aplicación permite a los usuarios escanear el código de barras de los productos y así ver su información, además de permitirles hacer y enviar fotografías y datos de los productos que faltan.

La aplicación para Android está desarrollada en Kotlin.

La aplicación clásica para iOS nunca se ha adaptado ni implementado para Open Pet Food Facts.

La nueva aplicación multiplataforma se desarrolla en Flutter/Dart.

Te invitamos a reutilizar el código (es de código abierto) y a ayudarnos a mejorarlo, para todos, en todo el planeta.
