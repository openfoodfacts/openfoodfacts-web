Por favor, leia os Termos e condições de uso e reutilização antes de reutilizar os dados.

Estamos muito interessados em saber em que são usados os dados do Open Pet Food Facts. Não é obrigatório, mas gostaríamos muito que nos dissesse como reutiliza os nossos dados para que possamos partilhá-los com a comunidade do Open Pet Food Facts.

Exportação de dados CSV

Os dados de todos os produtos, ou alguns dos produtos, podem ser descarregados no formato CSV (legível com o OpenOffice, Excel e muitos outros programas) através do formulário de pesquisa avançada.

Hiperligação
https://static.openpetfoodfacts.org/data/en.openpetfoodfacts.org.products.csv

A codificação do ficheiro é Unicode UTF-8. O caractere que separa os campos é <tab> (tabulação).

API JSON ao vivo

A API comporta-se 99% como o Open Food Facts, por isso muitos dos SDKs, os códigos para interagir com a API do Open Food Facts devem funcionar simplesmente alterando o endpoint de: openfoodfacts.org para openpetfoodfacts.org. O código não é atualizado há mais de um ano, por isso muitas das APIs mais recentes, como os painéis de conhecimento, ainda não estão disponíveis.

Também está disponível uma API JSON para ler os dados de um produto. Essa API é usada em particular na aplicação móvel Open Pet Food Facts para iPhone e Android.

Alteramos a API ao longo do tempo, mas temos o cuidado de manter a compatibilidade com versões anteriores. Por favor, diga-nos se a utilizar, para que possamos informá-lo sobre possíveis alterações.

URL para ler os dados de um produto: https://world.openpetfoodfacts.org/api/v0/product/[barcode].json

Exemplo: https://world.openpetfoodfacts.org/api/v0/product/3600550578041.json

  • API V1- Documentação antiga disponível na wiki (nota importante: a página está dividida em subpáginas para READ e WRITE).
  • API V2 - Abrangente, mas não compatível com OpenAPI - Contém casos de utilização para mostrar um produto e permitir que seus utilizadores adicionem os dados necessários para obter o Nutri-Score, a classificação NOVA, os aditivos e o Eco-Score.
  • API V2 - nova documentação OpenAPI - Trabalho a decorrer
  • API V3 - nova documentação OpenAPI - Trabalho a decorrer

Pode usar a API para casos de produção, desde que 1 chamada de API = 1 varredura real de um utilizador. Qualquer tentativa de sobrecarregar a base de dados pela API provavelmente será bloqueada, pois as exportações completas diárias estão disponíveis nesta página.

Além disso, enviar um cabeçalho HTTP com a sua chamada de API é uma boa maneira de entrarmos em contacto, caso haja qualquer problema.

Se não tiver certeza de nada ou tiver alguma dúvida (não existem perguntas absurdas), fique à vontade para perguntar no Slack :-)

Aplicação móvel Android e iPhone (ajuda necessária!)

O código atualmente implantado para a aplicação Open Pet Food Facts está disponível no GitHub (Android. Nós precisamos de ajuda para adaptar o novo código Flutter para alimentos para animais de estimação. A aplicação permite que os utilizadores digitalizem o código de barras dos produtos, visualizem as informações do produto e tirem e enviem fotos e dados de produtos que faltam.

A aplicação para Android é desenvolvida em Kotlin.

A aplicação clássica para iOS nunca foi adaptada nem implantada para alimentos de animais de estimação.

A nova aplicação multiplataforma é desenvolvida em Flutter/Dart.

Está convidado a reutilizar o código (é open source) e nos ajudar a melhorá-lo, para todos, no planeta inteiro.
