JSON API на живо

API се държи 99% като Open Food Facts, така че много от SDK, кодовете за взаимодействие с Open Food Facts API трябва да работят, като просто променят крайната точка от: openfoodfacts.org на openpetfoodfacts.org. Кодът не е актуализиран от повече от година, така че много от по-новите API като Knowledge Panels все още не са налични.

Наличен е и JSON API за четене на данните за продукт. Този API се използва по-специално в мобилното приложение Open Pet Food Facts за iPhone и Android.

Променяме API с течение на времето, но внимаваме да поддържаме обратна съвместимост. Моля, кажи ни дали го използваш, за да можем да ти кажем за потенциални промени.

URL за четене на данни за продукт: https://world.openpetfoodfacts.org/api/v0/product/[barcode].json

Пример: https://world.openpetfoodfacts.org/api/v0/product/3600550578041.json

  • API V1- Стара документация, налична в Wiki (Важна забележка: страницата е разделена на подстраници за ЧЕТЕНЕ и ПИСАНЕ).
  • API V2 - Изчерпателен, но не съвместим с OpenAPI - Съдържа случаи на използване за показване на продукт и позволява на вашите потребители да добавят данните, необходими за получаване на Nutri-Score, NOVA класификация, добавки и Eco-Score.
  • API V2 - нова документация за OpenAPI - Работа в процес
  • API V3 - нова документация на OpenAPI - Работа в процес

Добре дошли сте да използвате API за производствени случаи, стига 1 API извикване да е = на 1 реално сканиране от потребител. Всеки опит за сканиране на базата данни с помощта на API много вероятно ще бъде блокиран, тъй като пълните ежедневни износи са достъпни точно на тази страница.

Освен това изпращането на HTTP хедър с вашето API извикване е добър начин да се свържем, ако възникне някакъв проблем.

Ако не сте сигурни в нещо или имате някакъв въпрос (няма глупав въпрос), не се колебайте да го зададете на Slack :-)

Мобилно приложение за Android и iPhone (необходима е помощ!)

Кодът за мобилното приложение Open Pet Food Facts, което е внедрено в момента, е достъпно в GitHub (Android. Ние имаме нужда от помощ, за да адаптираме новия код на Flutter към храната за домашни любимци. Приложението позволява на потребителите да сканират баркода на продуктите, да преглеждат информацията за продукта и да правят и изпращат снимки и данни за липсващи продукти.

Приложението за Android е разработено в Kotlin.

Класическото приложение за iOS никога не е било адаптирано, нито използвано за храна за домашни любимци.

Новото междуплатформено приложение е разработено във Flutter/Dart.

Можеш да използваш повторно кода (той е с отворен код) и да ни помогнеш да го подобрим за всички, на цялата планета.
